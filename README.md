# Rotation-Angle Signatures in Self-Supervised Learning

This repository is a configurable independent reimplementation and reproducibility audit
of the experiments in the
[NeurReps 2025 paper](https://openreview.net/forum?id=85rdlgTS50):

> Paul Yan, Amy Saranchuk, and Michael Guerzhoy. “Response Patterns to Rotation Angle in
> a Rotation Pretext Task Vary Across Datasets and Architectures: An Observation and a
> Negative Result.” *Symmetry and Geometry in Neural Representations*, 2025.

It implements the 16×16 classification sweep, row/column curve prediction, the 3×3
medical segmentation sweep, the HoG/RBF-SVM shortcut test, and generation of Figures
1–4. Every long-running job is deterministic, independently resumable, and records its
configuration and software provenance.

## Important reproducibility boundary

The paper does not publish enough detail to make an exact numerical rerun possible from
the PDF alone. In particular, it omits the contrastive pretraining schedules, most
preprocessing choices, splits, seeds, projection heads, MLP widths, and raw medical
results. It also describes **921,600 independent contrastive pretraining jobs** for Figure
1, followed by 921,600 downstream MLP fits. A 200-epoch interpretation is on the order of
millions of GPU-hours.

The authors' public [`thesis`](https://github.com/paulyan678/thesis) repository does contain
the 256 original Figure 1 curves (921,600 points). This package fetches those curves at the
immutable commit `bf9d887337`, validates their complete 16×16×3,600 schema, and uses them
to reproduce Figure 1 and perform the Appendix C prediction experiment. They are labeled
**reference results**, not outputs regenerated by this implementation.

Accordingly:

- `configs/paper.yaml` is the faithful protocol plus explicit, documented assumptions for
  every choice the paper leaves open.
- `configs/smoke.yaml` exercises all three training paths on deterministic synthetic data.
- the generated Figure 1 can be checked immediately from the authors' published curves;
- newly trained curves and medical results are written separately under `outputs/`;
- no output is represented as an exact regeneration unless it was actually computed.

See [Paper gaps and assumptions](docs/PAPER_GAPS.md) for the full disclosure table.

## Installation

Python 3.10+ is required. A CUDA GPU is strongly recommended for any non-smoke training.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e '.[dev,medical]'
```

`requirements-lock.txt` records the exact environment used for the validation reported in
this repository. Install it before `-e .` when version-for-version replay is required.

## Reproduce the published Figure 1 and analyze its curves

```bash
# Downloads only the 256 CSVs from the pinned public source and validates all points.
rotation-patterns fetch-reference --config configs/paper.yaml

# Recreates the 16×16 result grid.
rotation-patterns figures --config configs/paper.yaml --only figure1

# Runs the Appendix C row/column prediction analysis on the real curves.
rotation-patterns prediction --config configs/paper.yaml
rotation-patterns figures --config configs/paper.yaml --only figure2
```

Artifacts are saved under `outputs/paper/figures/` and `outputs/paper/prediction/`.
The Appendix C reconstruction is an auditable negative result: its coherent shared
weight-vector interpretation does not recover the published Figure 2 values, while the
literal per-target wording would condition training on the label being predicted.
`summary.json` records digitized published means beside reconstructed means instead of
presenting the new violin plot as a match.

## Validate the training implementation

```bash
pytest
rotation-patterns smoke --config configs/smoke.yaml
```

The smoke profile exercises both SimCLR and MoCo v2, the six-layer downstream probe, a
MoCo-initialized U-Net, and the HoG/SVM baseline on small deterministic fixtures. Unit
tests separately cover reference parsing and prediction primitives; the pinned-reference
commands above validate Figures 1 and 2. Smoke metrics are implementation checks—not
substitutes for paper results.

## Run the full experiment grid

First read [Datasets](docs/DATASETS.md), place manually licensed datasets under `data/`,
and inspect every assumption in `configs/paper.yaml`.

```bash
rotation-patterns estimate --config configs/paper.yaml
rotation-patterns make-manifests --config configs/paper.yaml --shard-size 10000
```

Each CSV row is one independent, idempotent job. A single row can be executed with:

```bash
rotation-patterns run-task \
  --config configs/paper.yaml \
  --manifest outputs/paper/manifests/classification-0000.csv \
  --index 0
```

For Slurm, submit each manifest as an array:

```bash
mkdir -p logs
MANIFEST=outputs/paper/manifests/classification-0000.csv
N=$(($(wc -l < "$MANIFEST") - 2))
sbatch --array=0-"$N" scripts/slurm_array.sh configs/paper.yaml "$MANIFEST"
```

Completed rows are atomic JSON files. Re-running a completed row skips it unless `--force`
is used. Aggregate machine-readable results and render the figures with:

```bash
rotation-patterns aggregate --config configs/paper.yaml --experiment classification
rotation-patterns aggregate --config configs/paper.yaml --experiment segmentation
rotation-patterns aggregate --config configs/paper.yaml --experiment hog
rotation-patterns figures --config configs/paper.yaml
```

## Experiment map

| Paper result | Implementation | Output |
|---|---|---|
| Figure 1, 16×16 accuracy curves | fixed-angle SimCLR/MoCo v2 + frozen six-layer probe | `aggregate/classification.csv`, `figures/figure1.*` |
| Figure 2, signature prediction | Appendix C held-out-axis weighted similarity classifier | `prediction/trials.csv`, `figures/figure2.*` |
| Figure 3, 3×3 Dice + HoG | MoCo → U-Net and grouped HoG/RBF-SVM | `aggregate/{segmentation,hog}.csv`, `figures/figure3.*` |
| Figure 4, qualitative BraTS panel | stored test prediction at the configured example angle | `figures/figure4.*` |

## Scientific safeguards

- The unrotated and rotated views are the **only** contrastive augmentation.
- Every angle starts from a freshly constructed ImageNet-initialized encoder in the paper
  profile.
- Dataset splits, negative pairs, and job IDs are deterministic.
- Medical and HoG splits are grouped by source item to prevent paired-view leakage.
- Model-selection validation data is separate from the reported segmentation test set.
- `0°` and `360°` remain separate segmentation jobs because the paper explicitly includes
  both, even though they are the same geometric transform.
- L2 is implemented as **negative Euclidean distance**, because Appendix C takes an
  `argmax` but does not say how distance is converted to similarity.
- All choices not specified in the paper are listed, not hidden.
